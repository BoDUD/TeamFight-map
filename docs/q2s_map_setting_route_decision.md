# Q2s MapSetting Route Decision

Q2S closes the current Q2 `map_setting` spike phase and defines the next route. It is not a map-editing PR: it does not generate a mutated `map_setting`, does not install a runtime override, does not run the game, does not approve a third runtime probe, and does not mutate `chunked_binary`, `packed4_0`, or `packed4_1`.

## Current Status

```text
background visual override: proven
map_setting loader takeover: proven
map_setting byte-identical round trip: proven
two-byte chunked_binary runtime probes: loader pass, no semantic signal
packed4_0 0-7: strong static direction-code evidence
packed4_0 code15: cross-component bridge candidate, not semantic proof
packed4_1: profile-level node-class descriptor candidate
visual correlation: ambiguous
node_world_transform: unproven
unclassified/residual anchor route: closed
runtime_mutation_allowed: false
packed4_mutation_allowed: false
third_chunked_binary_runtime_probe_allowed: false
map_editing_allowed: false
```

Core decision:

```text
No, the project is not ready for gameplay map editing.
map_setting gameplay editing remains blocked pending a proven node/world transform and semantic field proof.
```

## Proven

## Closed Or Low-Value Routes


## Route B: Runtime Anchor / Instrumentation Spike

This route is for eventually enabling real gameplay map editing. It must remain separate from the visual-only deliverable.

Before gameplay editing can resume, Route B must prove:

```text
node_world_transform
field semantic proof
local reversible effect
```

Possible evidence surfaces:

```text
DLL / ModExtension ability to read entity positions
runtime screenshot + deterministic entity position anchors
debug overlay / camera transform
memory-safe process observation
other reliable runtime evidence
```

Route B must not mix in visual deliverable work and must not perform a new mutation until a specific independent anchor surface is reviewed.

## Recommendation

Prefer Route A first:

```text
visual-only deliverable
```

Reason: visual override is the only currently proven user-facing route. A LOL-like non-gameplay map skin can deliver value without pretending that unresolved `map_setting` semantics are solved.

Route B remains valid, but it should be a separate runtime-anchor/instrumentation spike, not a continuation of blind `map_setting` mutation.
